# Handoff: 캡춰 앱 (Screen Capture App)

## Overview
마우스로 화면의 사각형 영역을 지정해 캡춰하는 Windows 데스크탑 앱입니다. 핵심 기능:
- 마우스 드래그로 캡춰 영역 지정, 경계 핸들로 크기 조정
- 시작점·끝점·크기를 앱에서 숫자로 세부 조절
- **캡춰 매크로**: 특정 좌표를 클릭 → 대기 → 캡춰를 반복 실행
- 저장 경로 지정 (기본값: 바탕화면의 오늘 날짜 폴더)
- 캡춰 히스토리 보기

**타깃 환경: WPF (.NET / C# / XAML).** 이 문서는 WPF 앱으로 구현하는 것을 전제로 작성되었습니다.

## About the Design Files
이 번들의 `캡춰 앱.dc.html`은 **HTML로 만든 디자인 레퍼런스(시각·동작 프로토타입)**이며, 그대로 가져다 쓰는 프로덕션 코드가 아닙니다. 4개 화면이 하나의 캔버스에 나란히 배치되어 있습니다. 작업의 목표는 이 디자인을 **WPF/XAML 환경에서 재현**하는 것입니다 — XAML `Window`/`UserControl`, `Style`/`ResourceDictionary`, 데이터 바인딩(MVVM) 같은 WPF의 표준 패턴을 사용하세요. HTML/CSS의 flex·grid는 WPF의 `Grid`/`StackPanel`/`DockPanel`로, `border-radius`는 `Border.CornerRadius`로, hover 상태는 `Trigger`/`VisualState`로 옮기면 됩니다.

> `support.js`는 HTML 프로토타입을 브라우저에서 렌더링하기 위한 런타임일 뿐, 구현과 무관합니다. 무시하세요.

## Fidelity
**High-fidelity (hifi).** 색상·타이포·간격·상태가 모두 확정된 픽셀 단위 목업입니다. 아래 디자인 토큰의 정확한 hex/px 값을 그대로 사용해 재현하세요. 단 폰트는 웹 폰트(Pretendard) 기준이며, WPF에서는 Pretendard를 번들 폰트로 포함하거나 시스템 기본(예: Segoe UI / 맑은 고딕)으로 대체할 수 있습니다.

### Typography
- 폰트 패밀리: **Pretendard** (UI 전반). 숫자/좌표/파일명은 **모노스페이스** (`ui-monospace`, WPF에서는 `Consolas` / `Cascadia Mono`).
- 스케일: 타이틀바 13px/600 · 섹션 제목 16px/700 · 본문 12~13px · 라벨 11px/500~700(letter-spacing 0.04em) · 입력값 14px 모노 · 작은 메타 10~11px.

### Spacing & Shape
- 창 모서리: `border-radius: 11px` → `CornerRadius="11"`
- 카드/입력 필드: 8~10px 라운드
- 버튼: 높이 36px(보조) / 46px(주요 CTA), 8~10px 라운드
- 패널 패딩: 20~22px
- 그림자(창): `0 28px 64px -22px rgba(0,0,0,0.55)` — WPF `DropShadowEffect`로 근사

## Screens / Views

### 1. 메인 창 — 영역 지정 & 좌표 세부 조절
- **목적**: 캡춰 영역을 미리 보고 좌표를 숫자로 정밀 조정한 뒤 캡춰.
- **레이아웃** (좌→우 3분할 + 상단 타이틀바):
  - **타이틀바** (높이 46px): 앱 아이콘(앰버 18px 라운드 사각) + "캡춰 앱", 우측 최소화/최대화/닫기 시스템 버튼.
  - **네비 레일** (폭 78px, `#161619`): 아이콘+라벨 세로 메뉴 — 캡춰(활성, 좌측 3px 앰버 인디케이터) / 매크로 / 기록 / 설정(하단 고정).
  - **중앙**: 헤더("영역 캡춰" + 설명) + 우측 "새 영역 선택" 아웃라인 버튼. 그 아래 **캡춰 캔버스**(`#0F0F11`): 대각선 줄무늬 플레이스홀더 위에 앰버 선택 사각형(2px `#F2A33C`, 내부 `rgba(242,163,60,0.07)`, 바깥은 `box-shadow: 0 0 0 2000px rgba(8,8,10,0.5)`로 딤 처리). 선택 영역 좌상단에 치수 배지(`1024 × 640`), 8개 핸들(코너 4 + 변 중앙 4, 9px 앰버 사각). "경계 드래그로 크기 조정" 힌트 칩.
  - **우측 인스펙터** (폭 288px, `#161619`): "영역 좌표" 라벨 → 시작점(X/Y), 끝점(X/Y), 크기(W/H) 각각 2열 스피너 입력. 포커스된 필드는 앰버 보더 + `0 0 0 3px rgba(242,163,60,0.13)` 글로우. 구분선 아래 저장 형식 세그먼트(PNG 활성/JPG/BMP), 하단에 앰버 "캡춰" CTA(46px) + `PrtSc` 단축키 칩.
- **상호작용**:
  - "새 영역 선택" 클릭 → **앱 창 최소화**, 전체 화면 오버레이(화면 3) 표시.
  - 스피너 ▲▼ 또는 직접 입력으로 좌표 변경 시 캔버스 선택 사각형이 실시간 반영. 시작/끝점 ↔ 크기(W/H)는 상호 연동(끝점 = 시작점 + 크기).
  - 입력 필드 hover/focus, 버튼 hover 상태 존재.

### 2. 캡춰 매크로 설정
- **목적**: 좌표 클릭 → 캡춰 시퀀스를 반복 자동 실행하도록 설정.
- **레이아웃** (폭 472px 단일 패널, 세로 스택):
  - 타이틀바 "캡춰 매크로" + 타겟 아이콘.
  - **동작 순서** 섹션: 단계 카드 리스트. 각 카드 = 번호 뱃지(앰버 톤) + 제목 + 모노 상세 + 드래그 핸들(`⋮⋮`). 예: ①마우스 클릭 `X 640 · Y 360 · 대기 0.1초`, ②화면 캡춰 `지정 영역 1024 × 640`. 아래 "좌표 단계 추가" 점선 버튼(hover 시 앰버).
  - **반복 & 대기** 섹션: 2열 — 반복 횟수(스피너, 단위 "회", 예 `10`) / 클릭 후 대기(스피너, 단위 "초", **기본값 `0.1`**).
  - **저장 경로** 섹션: 폴더 아이콘 + 경로 텍스트(`바탕화면\2026-06-23`) + "폴더" 찾아보기 버튼. 아래 "오늘 날짜 폴더 자동 생성"(YYYY-MM-DD) + 토글 스위치(켜짐=앰버).
  - 하단: 앰버 "매크로 실행" CTA(▶ 아이콘).
- **상호작용**:
  - 단계 카드 드래그로 순서 변경, "좌표 단계 추가"로 단계 삽입(좌표는 화면에서 클릭 지정).
  - 토글 ON이면 저장 경로가 `바탕화면\{오늘 날짜}`로 자동 설정(경로 입력 비활성/연동).
  - "매크로 실행" → 각 단계 순차 실행 × 반복 횟수, 단계 사이 지정 대기시간 적용.

### 3. 화면 오버레이 — 드래그 선택 & 경계 크기 조정
- **목적**: 앱이 최소화된 상태에서 전체 화면 위 영역을 드래그로 선택/조정 후 캡춰.
- **레이아웃** (풀스크린): 전체 화면 캡춰 대상 위에 어두운 딤(`rgba(8,9,12,0.62)`), 선택 사각형만 밝게 뚫림(`box-shadow: 0 0 0 3000px` 기법).
  - 상단 중앙 힌트 바(반투명 blur): "드래그하여 영역 선택 · 경계를 드래그해 크기 조정 · Esc 취소".
  - 선택 사각형: 2px 앰버 보더, 치수 배지(`842 × 516`), 8개 핸들(10px). 현재 잡고 있는 핸들은 강조(`0 0 0 3px rgba(242,163,60,0.3)`). 커서 옆 좌표 태그(모노, 예 `962, 512`).
  - 선택 영역 하단에 **플로팅 툴바**: 앰버 "캡춰" 버튼 + 재선택/되돌리기 아이콘 + 구분선 + 취소(X, hover 빨강).
- **상호작용**:
  - 빈 곳 드래그 = 새 영역 그리기. 보더/핸들 드래그 = 크기 조정(코너=양방향, 변=한 방향). 내부 드래그 = 영역 이동.
  - 마우스가 경계 근처일 때 리사이즈 커서로 변경.
  - "캡춰" → 이미지 저장 후 **앱 창 복원**. Esc / 취소 → 저장 없이 앱 복원.

### 4. 캡춰 히스토리
- **목적**: 캡춰한 이미지 목록 확인.
- **레이아웃**: 메인 창과 동일한 셸(타이틀바 + 네비 레일, "기록" 활성). 콘텐츠: 헤더("캡춰 히스토리") + 우측 현재 저장 경로 칩 + 검색 필드. "오늘 · 2026년 6월 23일 · 8개" 그룹 라벨. 그 아래 **4열 썸네일 그리드**(gap 18px): 카드 = 16:10 썸네일(hover 시 앰버 보더) + 파일명(모노, 예 `캡춰_14-32-08.png`) + 메타("오후 2:32 · 248 KB").
- **상호작용**: 썸네일 hover 강조, 클릭 시 미리보기/열기(상세는 구현 재량). 검색으로 파일명 필터.

## Interactions & Behavior (요약)
- **새 영역 선택 → 최소화 → 오버레이**: 메인 창 "새 영역 선택"(또는 매크로 좌표 지정) 시 `Window.WindowState = Minimized`, 풀스크린 투명 오버레이 윈도우(`AllowsTransparency`, `WindowStyle=None`, `Topmost`)를 띄움. 캡춰/취소 시 오버레이 닫고 메인 창 복원.
- **전역 좌표 캡춰/클릭 자동화**: 화면 절대 좌표 기준. 마우스 이동·클릭 시뮬레이션은 Win32 `SendInput`/`mouse_event` P/Invoke, 화면 비트맵은 `Graphics.CopyFromScreen` 또는 `BitBlt` 사용. DPI 스케일링(좌표 ↔ 픽셀) 처리 필요.
- **애니메이션**: 버튼/카드 hover 색 전환 ~120ms ease. 핸들 잡기 글로우. 과한 모션 지양.
- **상태**: 선택 영역(StartX/Y, EndX/Y, W/H, 상호 연동), 저장 형식(PNG/JPG/BMP), 매크로 단계 리스트, 반복 횟수, 대기 시간(기본 0.1s), 저장 경로 + 오늘날짜 자동 토글, 히스토리 항목 리스트.

## State Management
MVVM 권장. 주요 ViewModel 속성:
- `CaptureRegion`: StartX, StartY, EndX, EndY, Width, Height (양방향 연동 — 끝점/크기 중 하나 변경 시 다른 값 재계산).
- `SaveFormat` (enum: Png/Jpg/Bmp).
- `MacroSteps` (ObservableCollection — 클릭 좌표/대기, 캡춰 단계), `RepeatCount`(기본 1+), `WaitSeconds`(기본 0.1).
- `SavePath`, `UseTodayDateFolder`(bool, 기본 true → `Desktop\yyyy-MM-dd`).
- `HistoryItems` (ObservableCollection — 썸네일, 파일명, 시각, 크기).

## Assets
- **아이콘**: 모두 인라인 SVG(라인 스타일, stroke-width 1.4~1.8)로 그려져 있음 — 캡춰(크롭 프레임), 매크로(타겟), 기록(시계), 설정(슬라이더), 재생/추가/검색/폴더/리사이즈 화살표 등. WPF에서는 동등한 벡터 아이콘 세트(예: Fluent System Icons, Lucide → XAML Path)로 대체 권장. 별도 래스터 이미지 없음.
- **폰트**: Pretendard(웹). 번들 포함 또는 Segoe UI / 맑은 고딕으로 대체.
- **썸네일/캡춰 미리보기**: 디자인에서는 줄무늬 플레이스홀더. 실제로는 캡춰된 비트맵.
